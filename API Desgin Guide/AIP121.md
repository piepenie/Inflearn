## AIP 121 요약
- API 가 제공할 자원 정의 : 데이터 알맹이가 무엇인지 명사(리소스)부터 확실히 챙기기 ex) Student, Book, File
- 관계 및 계층 구조 정의 : 명사끼리의 부모 자식 관계에 대해 파악
ex) Student(부모), Grade(자식). 단반향으로 갈끔하게 짜기.
- 스키마(데이터 구조) 정의 : Student 안에 이름(문자), 나이(숫자)가 들어가고, 해당 정보는 조희나 수정할 때 똑같아야함
- 메서드(동사) 정의(표준 동사 활용) : 자원 구조가 다 짜지면, 구글이 정해준 Create, Get, List, Update, Delete 안에서 최대한 해결할 것.
